DAO Aufrufe werden IMMER über die *DAOSyncs gemacht.
Dort wird versucht über das REMOTE DAO die Anfrage zu bedienen.
Sollte dies nicht klappen wird die Anfrage LOKAL verarbeitet.
    Dabei wird in den SyncService ein *Sync Model hinterlegt.
    Nach Zeit X wird geschaut ob der Client wieder Online ist und die gesammelten Anfragen werden
    REMOTE ausgeführt und der LOKALE Cache wird aktualisiert.

    Die ursprüngliche Anfrage an das DAOSync (es wird ein Listener in der Methode übergeben) wird dann
    LOKAL bedient und die onRequestSuccess des übergebenen Listeners wird aufgerufen.
Sollte die REMOTE Anfrage direkt klappen wird das Ergebnis an den übergebenen Listener
übergeben und gegenenfalls der LOKALE Cache geupdated.


SyncModel
    performSync
        Es wird die ID des Objektes übergeben, welches gesynced werden soll.
        Daher muss dies aus der LOKALEN DB geholt werden und auf den SERVER gespeichert werden.
        Dabei kann es z.B. sein das man ein Objekt auf dem Server neu anlegt und es ein andere ID als das lokale hat.
        Problem ist natürlich auch wenn die vom Server kommende ID bereits vergeben ist...
